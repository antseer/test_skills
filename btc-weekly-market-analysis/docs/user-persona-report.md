## 优化建议

1. **增加简报模式降低门槛** — 用户需理解 OI/MVRV 等专业概念才能解读完整报告 → 增加 `--mode=brief` 参数，输出仅含综合结论、短期方向和 3 个关键观察变量的精简版（约 300 字）→ 有效用户群从 5 万扩大至约 15 万

2. **ETF 背离信号置顶突出显示** — 高价值的 ETF 背离信号隐藏在 Step 3 中不够醒目 → 在综合结论章节顶部增加「关键警示信号」快速标识，高置信信号触发时以醒目格式置顶 → 交易员关键信号识别时间从 5-10 分钟降至 30 秒

3. **文档化外部数据降级策略** — 期权偏斜、Brent 原油等 4 类数据无 MCP 覆盖，影响报告完整性 → 在对应章节增加免费外部 API 快速接入指引（EIA 油价 API、Yahoo Finance SPX 等），提供 curl 示例 → 减少「数据缺失」困惑，专业用户可自行补充完整七维分析

4. **差异触发模式提升时效性** — 使用频率为每周 1 次，与 regime 监测的高频运行节奏错配 → 增加「差异触发」模式：宏观信号枚举值发生翻转时自动生成 100 字「信号更新摘要」并 Telegram 推送 → 实际有效触发频率从每周 1 次提升至每周 3-5 次

5. **增加社交媒体发布格式** — KOL 用户有直接发布周报需求，当前 Markdown 格式未针对 X 平台优化 → 增加 `--format=thread` 选项，将周报自动拆解为 5-7 条不超过 280 字的 Thread 格式 → KOL 用户从报告到发布的转化摩擦降低，复用率从每周 1 次提升至每周 2 次
