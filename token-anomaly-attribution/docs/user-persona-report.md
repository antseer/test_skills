## Skill 功能摘要

| 维度 | 内容 |
|------|------|
| 核心功能 | 代币价格大幅异动时运行 9 步结构化归因框架，从 8 个维度量化分析涨跌原因，输出持续性评级和操作建议 |
| 输入/输出 | 输入：代币符号（必填）+ 可选参数（方向、时间窗口、链、合约地址、阈值）；输出：结构化归因报告（Markdown 表格 + 持续性评级 + 操作建议方向） |
| 知识门槛 | 中高，需理解 OI、资金费率、market_beta、Smart Money、exchange_netflow 等概念 |
| 使用频率 | 事件驱动型高频（每次代币出现 >10% 异动时触发）；活跃市场环境下每周 3-8 次 |
| 决策类型 | 交易决策（chase / wait / exit）+ 研究分析（归因报告）+ 风控决策（持续性评级） |

## 优化建议

1. **降低认知门槛，增加快速模式** — 8 维度分析对新手认知负荷过重 → 增加 `--mode=quick` 参数，仅输出 3 个核心结论（方向判断 + 持续性 + 操作建议），详细维度按需展开 → 预期效果：将有效受众从中高级扩展至中级用户，提升 30-50% 覆盖范围

2. **半自动化非量化因素验证** — 安全事件、合作公告等 5 个关键归因维度无法自动获取，但往往是最重要的驱动因素 → 集成 WebFetch 步骤，自动抓取 DeFiLlama Hacks API + CryptoPanic + 项目官方 Twitter 最近 48 小时公告 → 预期效果：大幅提升事件驱动型异动的识别率，减少用户额外搜索步骤

3. **增强小市值代币分析路径** — Smart Money 和 holders 数据对小市值/新兴 DeFi 代币覆盖不完整，但这类代币往往最需要归因分析 → 当数据不可用时，增加替代信号：DeFiLlama TVL 变化率 + 社交热度飙升 + DEX 流动性变化，构建小市值专项路径 → 预期效果：提升 DeFi 用户画像的实际使用价值

4. **支持批量比较分析模式** — 当前仅支持单代币分析，研究分析师常需同时对比多个代币 → 增加 `symbols=[HYPE,ARB,OP]` 批量参数，输出多代币归因比较矩阵 → 预期效果：对研究分析师的价值从辅助工具升级为核心工作流，提升专业用户使用频率

5. **从被动触发转为主动预警** — 当前需用户察觉异动后手动触发，错失分析最佳窗口 → 与价格监控 skill 联动，代币触发 threshold_pct 时自动运行归因分析并推送；或增加每日关注列表扫描模式 → 预期效果：对时效性要求高的交易者价值显著提升，使用频率可提高 2-3 倍

## 方法论说明

本报告采用"推导 + 验证"方法：先从 PRD 和 SKILL.md 文档推导目标用户假设，再尝试用 ant_market_sentiment 社交数据进行交叉验证。

由于 ant_market_sentiment MCP 工具在当前执行环境不可用，社交验证层改为基于领域知识的推断（参考 CoinGlass、Nansen、Glassnode 等同类工具的公开用户规模数据）。

社交数据仅反映公开讨论，无法覆盖"沉默需求"（如机构投研等私密场景）。
