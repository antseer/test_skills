## 优化建议

1. **降低输出门槛 — 增加简报模式** — 画像发现：知识门槛较高，MVRV/NVT 等概念对中级用户不友好 → 增加 `brief_mode=true` 参数，直接输出"信号等级 + 一句话结论 + 最关键 3 项指标"，省略技术细节；在完整报告中为每个指标增加简短中文解释注释 → 预期效果：将可服务用户范围从"了解链上指标的进阶用户"扩展至"只关心结论的中长期投资者"，降低认知摩擦。

2. **明确代理指标局限性 — 增加原版对比说明** — 画像发现：Skill 是对原版"比特币泡沫指数"的代理实现，专业用户（画像 2）可能发现数值与 Woobull 原版不符而产生信任问题 → 在报告首部增加"代理指标与原版差异"区块，说明代理组合与原版的系统性偏差，并提供 Woobull / Glassnode MVRV 的参考链接 → 预期效果：提升报告透明度，减少专业用户的信任损失，将局限性转化为诚实沟通的优势。

3. **提升 ETF 流向数据可解读性** — 画像发现：ETF 流向存在 T+1 延迟且"止跌"判断边界模糊，专业分析师（画像 2）需要原始数据而非仅标签 → 在 ETF 模块增加：(1) 数据截止时间戳标注；(2) 输出具体的 7 日/30 日净流量数值；(3) 增加"连续 N 日趋势"字段 → 预期效果：提升对机构分析师的专业可信度，减少二次核查需求。

4. **优化事件驱动触发机制** — 画像发现：使用频率以事件驱动为主，非熊市期间用户可能遗忘该 Skill → 在 SKILL.md 触发词中增加"BTC 周线大幅收阴"类事件描述；建议与情绪监控类 Skill 形成联动，当恐贪指数跌入 fear 区间时自动推荐触发本 Skill → 预期效果：从"用户主动记忆触发"改为"事件驱动推送触发"，显著提升实际使用率。

5. **强化中文用户体验** — 画像发现：方法论来源为中文 KOL（@monkeyjiang），中文 crypto 社区对该框架有天然认知基础，触发词覆盖度可提升 → 在触发词中增加更多中文自然语言表达，如"现在是抄底的好时机吗"、"熊市底部来了吗"、"BTC还会再跌吗"；如有多语言需求，增加 `lang=en` 参数支持英文输出模式 → 预期效果：更精准服务推文原生受众（中文 crypto 社区），同时通过英文触发词覆盖国际用户。
