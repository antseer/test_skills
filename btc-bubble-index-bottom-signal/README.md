# btc-bubble-index-bottom-signal

比特币底部信号分析 — 用链上估值+情绪+技术面+ETF资金流综合判断BTC是否处于熊市底部区域

---

## 这个工具做什么？

你在持仓BTC或考虑加仓时，最难受的问题是：**现在还能买吗？这是底部还是下跌中途？**

这个工具帮你做以下三件事：

1. **拉取5类数据**：MVRV（链上持仓盈亏比）、NVT（链上结算量/市值）、市场恐慌指数、周线RSI（技术超卖信号）、ETF机构资金流向
2. **按权重打分**：每项指标达到历史底部条件时贡献对应分数，汇总为0-100的"底部信号综合得分"
3. **输出信号等级 + 风险收益比**：告诉你现在是Strong（强底）、Moderate（中度底部）、Weak（弱信号）还是No Signal，同时算出从当前价到你目标价的潜在涨幅倍数

方法论来源：加密分析师 @monkeyjiang，核心逻辑是"比特币泡沫指数跌至10附近 = 历史性底部区域，2022年至今无一次失效"。由于原版泡沫指数是第三方私有指标，本工具用Antseer MCP的链上数据进行近似还原。

---

## 怎么用？

```
/btc-bubble-index-bottom-signal
```

带参数调用（所有参数均可选）：

```
/btc-bubble-index-bottom-signal BTC --price_target_usd=200000 --time_range=4y --bubble_index_threshold=10
```

| 参数 | 说明 | 默认值 |
|------|------|--------|
| symbol | 分析资产（当前仅支持BTC） | BTC |
| price_target_usd | 你的中长期目标价（USD），用于计算风险收益比 | 200000 |
| time_range | 历史分位计算窗口（2y / 4y） | 4y |
| bubble_index_threshold | 底部阈值，低于此值视为底部区域 | 10 |

## 什么时候用？什么时候不适合？

**适合的场景：**
- BTC在过去1-3个月下跌30%以上，你想判断是否处于历史底部区域
- 每周定期巡检，跟踪各维度指标是否向底部方向收敛
- 宏观利空出尽后（如美联储暂停加息），验证市场情绪和链上指标是否共振
- 想用结构化框架代替"感觉"来做加仓决策

**不适合的场景：**
- 短线交易（本工具定位为周线级别，不适合日内或短线操作）
- 山寨币底部判断（链上历史分位数据不完整，无法等效分析）
- 期望精确预测底部价格点位（本工具给出的是区域判断，非精确预测）
- 宏观重大事件窗口期（如美联储政策急变、交易所暴雷），此时历史规律可能暂时失效

## 前置依赖

### 为什么需要装 MCP？

这个工具需要实时的链上和市场数据才能工作。MCP（Model Context Protocol）服务器是数据来源 — 把它理解成"给 Claude 接上数据管道"。不装的话，Claude 拿不到数据，分析就跑不起来。

### MCP 服务安装

#### Antseer MCP

**Claude Code (CLI)**
```bash
claude mcp add --transport http --scope user antseer https://mcp.antseer.com/mcp
```

**OpenClaw / Claw**

在设置页面添加 MCP 服务器，填写：
- 名称：`antseer`
- URL：`https://mcp.antseer.com/mcp`
- 传输类型：`http`

**OpenCode**

在配置文件 `opencode.json` 中添加：
```json
{
  "mcpServers": {
    "antseer": {
      "type": "http",
      "url": "https://mcp.antseer.com/mcp"
    }
  }
}
```

**通用 MCP 客户端**

任何支持 MCP 协议的客户端均可接入，核心参数：
- MCP 端点：`https://mcp.antseer.com/mcp`
- 传输类型（Transport）：`http`
- 作用域（Scope）：`user`（推荐，跨项目共享）

安装完成后，**重启你的 Agent 客户端**以激活 MCP 服务。
