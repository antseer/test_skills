## 优化建议

### 1. 内置 Watchlist 模板，降低冷启动门槛

**画像发现**: 主观多头交易员对"应该分析哪些代币"没有现成参考，首次使用时需自行构建列表，阻力较大。

**具体改动**: 在 SKILL.md 中增加 3-5 个预设 watchlist（ai-narrative、defi-bluechip、top20-altcoin），
支持 `/token-resilience-screener --watchlist=ai-narrative` 语法。

**预期效果**: 新用户可在10秒内触发第一次有意义的分析，大幅提升首次使用成功率和留存。

---

### 2. 增加 JSON 结构化输出模式，覆盖量化策略师

**画像发现**: 量化策略师需要程序化消费韧性信号，对话式 Markdown 输出无法满足需求（该画像当前为"需校准"状态）。

**具体改动**: 增加 `--output_format=json` 参数，输出包含 tier、resilience_score、avg_daily_return、
excess_return_mean、best_day、narrative_tags 字段的结构化 JSON。

**预期效果**: 将 Skill 扩展为可程序化消费的上游数据接口，覆盖量化策略师画像，解锁与 OI/资金费率联动的场景。

---

### 3. 增加方法论内嵌说明，提升分析透明度

**画像发现**: 韧性得分公式（0.6×抗跌超额 + 0.4×跟涨超额）对中级用户不够直观，容易质疑分数可信度。

**具体改动**: 在 Step 6 周报末尾增加 3-5 行白话方法论简注；同时在每个代币条目下展示"抗跌超额 X/Y 天，跟涨超额 X/Y 天"原始依据。

**预期效果**: 提升周报可信度，减少用户对黑盒分数的疑虑，降低研究员人工 review 时间。

---

### 4. 文档化自动化集成路径，提升高频使用

**画像发现**: 大盘急跌时的应急韧性筛查是比每周定时更高价值的触发场景，当前文档未描述如何与监控系统集成。

**具体改动**: 在 Entry Conditions 中增加"自动化集成说明"章节，提供与 regime_engine 联动的示例配置
（BTC 7日跌幅触发 Z-score 阈值时自动执行 Skill）。

**预期效果**: 将 Skill 从手动周报工具升级为市场压力感知自动化节点，提升使用频率和用户粘性。

---

### 5. 明确快速近似降级模式，解决外部 API 依赖断点

**画像发现**: Skill 核心数据源依赖 CoinGecko 历史 API（外部调用），在 MCP 工具链中存在断点，可能导致首次执行失败。

**具体改动**: 在 Features/数据来源中明确区分精确模式（CoinGecko 历史 API，任意日期范围）和快速近似模式
（ant_spot_market_structure 7d 字段，固定最近7日），将近似模式作为默认降级路径。

**预期效果**: 提升首次执行成功率，减少因 API 速率限制导致的失败中断。

---

### 6. 增加口语化中文触发词，扩大可发现性

**画像发现**: 冷话题分析表明 Skill 缺乏自发发现路径，中文用户偏向口语化描述而非专业术语。

**具体改动**: 在 SKILL.md description 中补充触发词："哪些币这周抗跌"、"大跌周谁最强"、"这周最强的币"、"哪些代币跌得少"。

**预期效果**: 覆盖不熟悉"韧性分析"术语的中级交易员群体，降低首次触发门槛。
