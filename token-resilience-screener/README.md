# token-resilience-screener

代币韧性周报分析器 — 用相对强度数据找出大盘承压周内「涨时超涨、跌时超抗跌」的领涨候选标的

---

## 这个工具做什么？

你每周看着大盘一片绿，但总有几个币跌得比别人少、甚至逆势涨。这些币就是下周最可能率先反弹的候选。

这个工具帮你系统性地找出它们：

1. **拉数据**：从 CoinGecko 历史 API 拉取你指定代币（和基准如 ETH/BTC）在分析周期内每一天的收盘价
2. **算相对强度**：对比每个代币和基准在「上涨日」「下跌日」的表现差距，计算0到1的「韧性得分」
3. **分三级**：T1（周均正收益 + 抗跌）/ T2（跌幅远小于基准）/ 弱势警示（跌幅超过基准 2.5 倍），按得分降序排列
4. **出周报**：生成带逐日涨跌幅矩阵 + 一句话点评 + 下周重点标的的 Markdown 报告，可直接发布

---

## 怎么用？

### 基础调用

```
/token-resilience-screener tao kaito hype injective story drift 2026-03-21 2026-03-27
```

参数说明：
- 第1至N个词（CoinGecko ID 格式）：要分析的代币列表，必填
- 倒数第二个参数：分析起始日（YYYY-MM-DD），必填
- 最后一个参数：分析结束日（YYYY-MM-DD），必填

### 完整参数示例（含可选项）

```
/token-resilience-screener \
  symbols=["tao","kaito","hype","injective","story","drift"] \
  start_date="2026-03-21" \
  end_date="2026-03-27" \
  benchmark="ethereum" \
  vs_secondary="solana" \
  t1_min_avg=0.0 \
  t2_max_drawdown_ratio=0.5 \
  weak_drawdown_ratio=2.5
```

| 参数 | 必填 | 默认值 | 说明 |
|------|------|--------|------|
| symbols | 是 | — | 代币列表（CoinGecko ID，如 "tao" 不是 "TAO"） |
| start_date | 是 | — | 分析起始日 YYYY-MM-DD |
| end_date | 是 | — | 分析结束日 YYYY-MM-DD |
| benchmark | 否 | "ethereum" | 主基准，填 CoinGecko ID |
| vs_secondary | 否 | "solana" | 副基准，用作横向参照 |
| t1_min_avg | 否 | 0.0 | T1 门槛：周均日涨幅须达到此值（%） |
| t2_max_drawdown_ratio | 否 | 0.5 | T2 门槛：跌幅不超过基准均值的 N 倍 |
| weak_drawdown_ratio | 否 | 2.5 | 弱势触发倍数：跌幅超过基准 N 倍则警示 |

## 什么时候用？什么时候不适合？

### 适合用的场景

- 大盘单周跌幅超过3%，想快速找出哪些代币结构最好
- 每周日晚或周一早上，做上一周的复盘筛查
- 在已有 watchlist 的基础上，做相对强度排名（减少主观偏差）
- 结合宏观叙事，识别板块轮动中的领涨板块（AI > DePIN > DeFi 等）

### 不适合的场景

- 你还没有 watchlist，想让工具帮你从全市场找代币：不行，symbols 必须由你提供
- 你想判断"这个币下周涨不涨"：韧性是回顾性指标，不做预测
- 你想分析链上聪明钱流向、持仓结构：这个工具只看价格相对强度，链上数据需另外的工具
- 分析窗口超过30天：受 CoinGecko API 速率限制，长周期分析效率低
- 上市不足7天的新代币：历史数据不足，韧性得分无意义

## 前置依赖

### 为什么需要装 MCP？

这个工具用 Antseer MCP 获取市场情绪和叙事标签数据（Step 5，可选增强）。核心历史日线数据来自 CoinGecko 公开 API（无需 MCP），但要让分析结果附上「AI叙事」「DePIN板块」这类叙事标签，需要接入 Antseer MCP。把它理解成"给 Claude 接上加密市场的情绪数据管道"。

### Ant-on-Chain MCP（情绪数据，可选）

**Claude Code (CLI)**
```bash
claude mcp add --transport http --scope user ant-on-chain-mcp https://ant-on-chain-mcp.antseer.ai/mcp
```

**OpenClaw / Claw**

在设置页面添加 MCP 服务器，填写：
- 名称：`ant-on-chain-mcp`
- URL：`https://ant-on-chain-mcp.antseer.ai/mcp`
- 传输类型：`http`

**OpenCode**

在配置文件 `opencode.json` 中添加：
```json
{
  "mcpServers": {
    "ant-on-chain-mcp": {
      "type": "http",
      "url": "https://ant-on-chain-mcp.antseer.ai/mcp"
    }
  }
}
```

**通用 MCP 客户端**

任何支持 MCP 协议的客户端均可接入，核心参数：
- MCP 端点：`https://ant-on-chain-mcp.antseer.ai/mcp`
- 传输类型（Transport）：`http`
- 作用域（Scope）：`user`（推荐，跨项目共享）

安装完成后，**重启你的 Agent 客户端**以激活 MCP 服务。

不安装 Antseer MCP 也可以正常运行——只是叙事标签步骤会跳过，其他价格相对强度分析功能完全不受影响。
