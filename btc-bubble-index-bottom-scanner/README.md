# btc-bubble-index-bottom-scanner

BTC 底部信号扫描 — 用链上估值 + 技术面 + ETF 资金流判断比特币是否处于熊市底部区域

---

## 这个工具做什么？

你在市场大跌或熊市低迷期想知道："现在的 BTC 价格是真的到底了，还是还会继续跌？"

这个工具帮你同时检查四个维度，给出一个"底部信号强度评级"：

1. **链上估值**（MVRV 和 NVT 比率）— 用区块链数据算出"市场愿意为每一块钱链上价值支付多少钱"，历史上熊市底部这个数字会跌到极低位置
2. **技术面**（RSI 和 MACD）— 看价格是否进入超卖区域、是否出现底背离信号
3. **ETF 资金流**（机构资金动向）— 如果大机构在抄底买入，说明专业资金认可这个价位
4. **市场情绪**（恐慌贪婪指数）— 历史底部通常伴随极端恐慌情绪，这时候普通人在割肉，聪明钱在买入

最后输出综合评级（强底部信号 / 弱底部信号 / 非底部区间），并附上不同入场价格对应的潜在收益倍数对比表。

---

## 怎么用？

直接在对话框输入：

```
/btc-bubble-index-bottom-scanner
```

也可以带参数定制分析：

```
/btc-bubble-index-bottom-scanner symbol=BTC bottom_threshold=10 price_targets=[200000,300000]
```

**参数说明：**

| 参数 | 说明 | 默认值 | 示例 |
|------|------|--------|------|
| `symbol` | 分析目标（当前仅支持 BTC） | BTC | BTC |
| `bottom_threshold` | 泡沫指数底部阈值（低于此值视为底部） | 10 | 10 |
| `lookback_years` | 历史数据回溯年数 | 4 | 4 |
| `price_targets` | 目标价格列表（USD），用于算收益倍数 | [200000] | [200000, 300000] |
| `time_range` | 技术指标分析时间范围 | 1y | 1y |

所有参数都是可选的，默认就能跑。

## 什么时候用？什么时候不适合？

**适合的场景：**
- 熊市低迷期，想知道是否已到底部区域
- 行情单日大跌 10% 以上，需要快速判断是否触底
- 定期（每周或每月）追踪底部信号变化趋势
- 评估中长线入场的收益风险比

**不适合的场景：**
- 短线（日内）交易决策 — 这个工具是给中长线投资者用的，不是用来判断今天涨还是跌的
- 山寨币分析 — MVRV/NVT 数据覆盖有限，当前仅支持 BTC
- 需要精确底部价格或时间的场景 — 工具只判断"是否处于底部区域"，不预测具体数字
- 黑天鹅事件判断 — 监管打压、交易所暴雷等结构性风险无法通过指标预测

**重要提示：** 原始"比特币泡沫指数"数据（推文作者使用的核心指标）需要在 CryptoQuant 或 Glassnode 上手动查询"BTC Bubble Index"或"MVRV Z-Score"进行对照验证。本工具使用 MVRV 作为其最佳代理指标，两者结构高度相似但并非完全等价。

## 前置依赖

### 为什么需要装 MCP？

这个工具需要实时的链上数据和市场数据才能工作。MCP（Model Context Protocol）服务器是数据来源 — 把它理解成"给 Claude 接上数据管道"。不装的话，Claude 拿不到链上数据，分析就跑不起来。

### MCP 服务安装

#### Antseer MCP（链上数据服务）

本 Skill 依赖以下 Antseer MCP 工具：
- `ant_token_analytics` — MVRV、NVT 链上估值数据
- `ant_spot_market_structure` — 实时价格数据
- `ant_market_indicators` — RSI、MACD 技术指标
- `ant_etf_fund_flow` — BTC ETF 资金流向
- `ant_market_sentiment` — 市场情绪评分

**Claude Code (CLI)**

```bash
claude mcp add --transport http --scope user antseer https://mcp.antseer.com/v1
```

**OpenClaw / Claw**

在设置页面添加 MCP 服务器，填写：
- 名称：`antseer`
- URL：`https://mcp.antseer.com/v1`
- 传输类型：`http`

**OpenCode**

在配置文件 `opencode.json` 中添加：

```json
{
  "mcpServers": {
    "antseer": {
      "type": "http",
      "url": "https://mcp.antseer.com/v1"
    }
  }
}
```

**通用 MCP 客户端**

任何支持 MCP 协议的客户端均可接入，核心参数：
- MCP 端点：`https://mcp.antseer.com/v1`
- 传输类型（Transport）：`http`
- 作用域（Scope）：`user`（推荐，跨项目共享）

安装完成后，**重启你的 Agent 客户端**以激活 MCP 服务。
