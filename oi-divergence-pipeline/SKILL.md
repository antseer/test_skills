# OI 背离自动狙击管线

自动化做空目标发现管线：从合约涨幅榜发现异常拉升 → OI 预筛 → 控盘检测 → 深度背离分析 → 综合评分排名。

**理论来源**: @wuk_Bitcoin 崩盘模型 + 余烬 (@EmberCN) 控盘检测方法
1. 价格被托着往上走，合约持仓量连续降低 — 崩盘前的预谋行为
2. 主力高位平多，托住价格找对手盘，让散户进来做多
3. 有了对手盘后叠加空单，布局完成后撤掉托价订单砸盘
4. 验证前提：通过链上 Holder 集中度确认是否存在控盘实体

## 输入

`$ARGUMENTS` = 可选参数。不传参则使用默认阈值直接运行。

支持的参数格式：
- 空（默认）：4H>8% 或 24H>15%
- 百分数如 `10%`：自定义 4H 涨幅阈值（24H 阈值 = 4H 阈值 × 1.8）
- `low`：低阈值模式（4H>5%, 24H>10%），适合熊市
- `high`：高阈值模式（4H>12%, 24H>25%），适合牛市

## 管线概览

```
Phase 1: 目标发现 ──→ Phase 2: OI 预筛 ──→ Phase 3: 控盘检测 ──→ Phase 4: 深度背离 ──→ Phase 5: 评分输出
  100+ 币种           ~8 候选            ~4 候选             ~3 候选            Top 5 推荐
  (涨幅筛选)          (排除正常加仓)      (验证控盘前提)       (完整背离分析)       (综合排名)
```

## Phase 1: 目标发现

**目的**：从合约市场涨幅榜找到异常拉升的币种。

### Step 1.1 — 并行拉取数据

同时调用以下 2 个查询：

**查询 A — 涨幅排行：**
- 工具: `ant_futures_market_structure`
- query_type: `futures_price_change`
- symbol: 不传（返回所有币种的多时间段涨跌幅）

**查询 B — 市场快照：**
- 工具: `ant_futures_market_structure`
- query_type: `futures_market_snapshot`
- per_page: `100`
- page: `1`

### Step 1.2 — 筛选逻辑

解析 `$ARGUMENTS` 确定阈值：
- 默认：4H_THRESHOLD = 8%, 24H_THRESHOLD = 15%
- 自定义百分数：4H_THRESHOLD = 用户值, 24H_THRESHOLD = 用户值 × 1.8
- `low`：4H_THRESHOLD = 5%, 24H_THRESHOLD = 10%
- `high`：4H_THRESHOLD = 12%, 24H_THRESHOLD = 25%

**候选条件**（满足任一）：
- 4H 涨幅 > 4H_THRESHOLD
- 24H 涨幅 > 24H_THRESHOLD
- 1H 涨幅 > 5% 且该币种在涨幅榜前 20

**排除条件**：
- BTC、ETH（大盘币种背离含义不同，不适用此模型）
- 24H 成交量 < $1M（太薄无法做空）
- 如果候选超过 15 个，按 4H 涨幅降序截取 Top 15

### Step 1.3 — 输出中间结果

在报告中简要列出 Phase 1 结果：
```
Phase 1 完成：从涨幅榜筛出 N 个异常拉升币种
```

列出每个候选的 symbol、1H/4H/24H 涨幅、24H Vol。

## Phase 2: OI 快速预筛

**目的**：快速排除 OI 同步上涨的币种（正常多头加仓，不符合崩盘模型）。

### Step 2.1 — 并行查询所有候选币的 OI

对 Phase 1 每个候选币种，**并行调用**：
- 工具: `ant_futures_market_structure`
- query_type: `futures_oi_aggregated`
- symbol: 候选币种（大写）
- interval: `1h`
- limit: `12`（只看最近 12 小时，快速判断趋势）

### Step 2.2 — 快速判断逻辑

计算最近 12 根 1H K 线的 OI 变化趋势（首尾对比）：

| OI 变化 | 判断 | 动作 |
|---------|------|------|
| OI 上涨 > 5% | 多头正常加仓，不符合崩盘模型 | 排除 |
| OI 基本持平（-2% ~ +5%） | 可疑，需深入分析 | 保留 |
| OI 下降 > 2% | 强可疑信号，价格涨但 OI 降 | 保留（优先） |

### Step 2.3 — 输出中间结果

```
Phase 2 完成：N 个币种通过 OI 预筛（排除了 M 个 OI 同步上涨的币种）
```

如果 0 个通过，直接跳到输出阶段，报告"本轮无异常"。

---

## Phase 3: 控盘检测（精简版）

**目的**：验证崩盘模型的前提条件——是否存在庄家控盘迹象。

> 注意：这是精简版控盘检测，不做完整地址聚类（那需要 5+ 次 API/币）。
> 用户可对高评分币种手动运行 `/whale-control` 做深入分析。

### Step 3.1 — 获取代币合约地址

对每个候选币种，调用：
- 工具: `ant_meme`
- query_type: `search_pairs`
- query: 币种符号

从结果中提取 baseToken.address 和 chainId。如果找不到（纯 CEX 币种），跳过链上分析。

### Step 3.2 — 并行查询 Holder + 鲸鱼仓位

对每个候选，**并行调用**：

**查询 A — Holder 集中度：**（如果有合约地址）
- 工具: `ant_token_analytics`
- query_type: `holders`
- token_address: 合约地址
- chain: 链名称
- pagination: `{"page": 1, "per_page": 20}`

**查询 B — Hyperliquid 鲸鱼仓位：**
- 工具: `ant_perp_dex`
- query_type: `perp_dex_position_by_coin`
- symbol: 币种符号（大写）

### Step 3.3 — 控盘判断

**Holder 集中度（来自查询 A）：**

从 Top 20 Holder 中排除交易所地址、合约地址（LP 池、锁仓）、销毁/零地址后，计算 EOA 持仓占比：

| Top 5 EOA 持仓 | 控盘评级 | 控盘分 |
|----------------|---------|--------|
| > 65% | 庄家盘 | 100 |
| 40-65% | 高风险 | 75 |
| 20-40% | 关注 | 40 |
| < 20% | 健康 | 10 |

**鲸鱼仓位（来自查询 B）：**

检查 Hyperliquid 上是否有大户集中做多。有大户集中做多 = 为后续砸盘提供动机验证。

### Step 3.4 — 筛选与降级

**保留条件**（满足任一）：
- 控盘评级 >= "关注"（Top 5 EOA > 20%）
- 鲸鱼持仓显著（有大户集中做多）
- Phase 2 的 OI 降幅 > 5%（即使无链上数据，OI 大幅下降本身就是强信号）

**降级策略**：
- 无链上数据（search_pairs 找不到）→ 仅用鲸鱼仓位判断
- 无鲸鱼数据 → 仅用 OI 降幅 > 5% 通过
- 两者都无 → 该币种控盘分 = 25（中性假设），不排除但降低最终权重

### Step 3.5 — 输出中间结果

```
Phase 3 完成：N 个币种通过控盘筛选
```

如果 0 个通过，跳到输出阶段。

## Phase 4: 深度 OI 背离分析

**目的**：对通过前三层筛选的币种执行完整 OI 背离分析（复用 oi-divergence 核心逻辑，增加爆仓维度）。

### Step 4.1 — 并行拉取 4 维数据

对每个候选币种，**并行调用**以下 4 个查询：

**查询 A — 24 根 1H OI：**
- 工具: `ant_futures_market_structure`
- query_type: `futures_oi_aggregated`
- symbol: 币种（大写）
- interval: `1h`
- limit: `24`

**查询 B — 24 根 1H 价格：**
- 工具: `ant_futures_market_structure`
- query_type: `futures_price_history`
- symbol: 币种（大写）
- exchange: `binance`
- interval: `1h`
- limit: `24`

如果 binance 无数据，依次尝试 `bybit`、`okx`。

**查询 C — 24 根 1H 资金费率：**
- 工具: `ant_futures_market_structure`
- query_type: `futures_funding_rate_oi_weight`
- symbol: 币种（大写）
- interval: `1h`
- limit: `24`

**查询 D — 24 根 1H 爆仓数据：**
- 工具: `ant_futures_liquidation`
- query_type: `futures_liquidation_aggregated`
- symbol: 币种（大写）
- interval: `1h`
- limit: `24`

### Step 4.2 — 数据对齐

将 OI、价格、资金费率、爆仓数据按时间戳对齐。只分析所有数据源都有数据的时间段。

### Step 4.3 — 背离分析

重点分析最近 6-12 根 1H K 线：

**价格趋势判断：**
- 计算价格 close 的变化方向
- 是否在走高（higher highs / higher closes）

**OI 趋势判断：**
- 计算 OI close 的变化方向
- 是否在持续下降

**背离判定：**

| 价格 | OI | 判定 |
|------|-----|------|
| 上涨 | 下降 | 背离成立（做空信号） |
| 下跌 | 下降 | 正常平仓，不算背离 |
| 上涨 | 上涨 | 多头加仓，不算背离 |
| 下跌 | 上涨 | 空头加仓，逆向信号 |

**背离强度评级：**

| 等级 | 连续 K 线数 | OI 降幅 | 背离分 |
|------|-----------|---------|--------|
| 强 | >= 6 根 | > 5% | 100 |
| 中 | 3-5 根 | 2-5% | 60 |
| 弱 | 2-3 根 | < 2% | 30 |

加权调整：连续 K 线每多 1 根（超过基准），背离分 +5，上限 +20。

**辅助确认信号：**
- 资金费率 > 0.01%：多头拥挤确认（加分）
- 资金费率持续为正但 OI 在降：主力在平多（加分）
- 近期爆仓以多头为主：已有人被清算（加分）
- 爆仓量突然放大：市场脆弱性确认（加分）

### Step 4.4 — 做空点位计算

仅在背离强度 >= "中" 时计算：

- **入场区间**: 当前价格附近，或最近高点回踩位
- **止损位**: OI 开始下降时对应的价格高点上方 1-2%
- **目标位 1**: 最近 24H 价格低点
- **目标位 2**: OI 下降起始点对应的价格位
- **盈亏比**: (入场中位 - 目标位 1) / (止损位 - 入场中位)

### Step 4.5 — 筛选

仅保留背离强度 >= "中" 的币种进入 Phase 5。

## Phase 5: 综合评分与输出

**目的**：多维信号汇总为综合评分，排序输出。

### Step 5.1 — 评分公式

```
总分 = 背离分(40%) + 控盘分(25%) + 资金费率分(15%) + 涨幅异常分(10%) + 流动性分(10%)
```

**各维度评分规则：**

| 维度 | 条件 | 分值 |
|------|------|------|
| **背离分** (0-100) | 强=100, 中=60, 弱=30 | 权重 40% |
| | 连续 K 线每多 1 根 +5（上限 +20） | |
| **控盘分** (0-100) | 庄家盘(>65%)=100, 高风险(40-65%)=75, 关注(20-40%)=40, 健康(<20%)=10 | 权重 25% |
| | 无数据=25（中性假设） | |
| **资金费率分** (0-100) | >0.05%=100, 0.03-0.05%=80, 0.01-0.03%=50, <0.01%=20 | 权重 15% |
| **涨幅异常分** (0-100) | 24H>30%=100, 20-30%=80, 15-20%=60, 10-15%=40, <10%=20 | 权重 10% |
| **流动性分** (0-100) | Vol<$5M=80, $5-20M=60, $20-100M=40, >$100M=20 | 权重 10% |

> 流动性分越低分越高：低流动性更容易被控盘砸盘。

### Step 5.2 — 排序与截断

- 按总分降序排列
- 总分 < 40 的币种不输出
- 最多输出 Top 5

## 降级处理

每个 Phase 的数据缺失不阻塞整个管线，而是降级处理：

| 场景 | 降级策略 |
|------|----------|
| `futures_price_change` 返回空 | 终止管线，报告"数据源异常" |
| 某币种 OI 数据为空 | 跳过该币种，不进入 Phase 3 |
| `search_pairs` 找不到合约地址 | 跳过链上 Holder 分析，仅用鲸鱼仓位 |
| `holders` 返回空 | 控盘分 = 25（中性），不排除 |
| `perp_dex_position_by_coin` 无数据 | 无鲸鱼辅助信号，仅用 Holder 数据 |
| 价格数据 binance 无 | 依次尝试 bybit、okx |
| `futures_liquidation_aggregated` 无数据 | 跳过爆仓维度分析 |
| API 超时 | 重试 1 次，仍失败则跳过该查询 |

## 注意事项

- 控盘比例高不一定恶意（可能是未解锁代币、DAO 金库、合规做市商库存）
- 避免"骗局""诈骗"等定性词汇，使用"疑似""可能""推测"
- 每份报告必须包含免责声明和风险提示
- 大盘系统性暴涨时候选会过多，阈值可通过参数调高
- 建议搭配定时任务使用：`scheduled task，每 4 小时运行 /oi-divergence-pipeline`
