# cex-volume-anomaly-signal

CEX 换手量异常底部信号分析器 — 用 CEX 交易量数据检测历史极端换手区间，识别资产底部出清信号

---

## 这个工具做什么？

你在市场大幅下跌的时候，想知道"这次的抛售力度跟 FTX 暴雷、3AC 崩盘那时候比，到底严不严重？"

这个工具帮你：

1. 拉取目标资产在 CEX（中心化交易所，比如币安）上的近期换手量，并用 ETF 资金流数据做稀释修正（把机构通过 ETF 渠道买卖的量还原进来，让对比更公平）
2. 把当前换手量跟历史压力事件期间（3AC 暴雷、FTX 暴雷、LUNA 崩盘等）的数据逐一对比，算出"当前压力等于历史几成"
3. 验证有没有净流出（真的在抛售出清，还是高频交易刷出来的假量）、交易所储备是否同步下降
4. 对比跨资产底部时序（比如历史上 ETH 是否先于 BTC 触底），推断当前资产在周期中的位置
5. 汇总成 0-100 分的底部信号评分，给出强/中/弱/无的信号等级和操作建议

方法论来源：交易员 @Michael_Liu93（2026-02-23 推文），原始分析对象为 ETH。

---

## 怎么用？

```
/cex-volume-anomaly-signal ETH
```

完整参数示例：

```
/cex-volume-anomaly-signal ETH exchange=binance lookback_days=7 benchmark_events=["3AC","FTX","LUNA"] etf_adjustment=true comparison_asset=BTC
```

**参数说明：**

| 参数 | 必填 | 默认值 | 说明 |
|------|------|--------|------|
| symbol | 是 | -- | 目标资产，如 ETH、BTC、SOL |
| exchange | 否 | binance | 观察哪个交易所的数据（建议 binance，数据最可信）|
| lookback_days | 否 | 7 | 计算当前换手量的时间窗口（天）|
| benchmark_events | 否 | ["3AC","FTX"] | 用哪些历史事件做对比基准；可选：3AC、FTX、LUNA、2020-3.12 |
| etf_adjustment | 否 | true | 是否做 ETF 稀释修正（仅 ETH/BTC 有效，其他资产自动跳过）|
| comparison_asset | 否 | BTC | 跨资产时序对比的参照标的 |
| time_range | 否 | 2022-01-01 至今 | 历史数据范围 |

## 什么时候用？什么时候不适合？

**适合使用的场景：**
- 市场出现大幅下跌（近 30 天跌幅 >30%），想量化这次抛售到底有多严重
- 听说某资产交易量创新高，想知道是真的出清还是高频刷量
- 想复刻 @Michael_Liu93 的 CEX 换手量底部分析方法，快速套用到任意主流资产
- 做中长线布局前，想用量化数据支撑入场时机判断

**不适合使用的场景：**
- 小市值代币（流动性不足，量能信号失真）
- 新上线资产（无足够历史数据做对比）
- 需要实时追踪暗池/OTC 交易量（本工具不覆盖）
- 需要精确的入场价格或止损点位（本工具不提供价格预测）
- 突发性黑天鹅事件发生后的即时判断（历史类比可能失效）

## 前置依赖

### 为什么需要装 MCP？

这个工具需要实时的 CEX 市场数据、ETF 资金流数据和链上资金流向数据才能工作。MCP（Model Context Protocol）服务器是数据来源——把它理解成"给 Claude 接上数据管道"。不装的话，Claude 拿不到数据，分析就跑不起来。

本工具使用的 Antseer MCP 工具：
- `ant_spot_market_structure`（现货市场结构数据）
- `ant_market_indicators`（市场技术指标：taker flow、RSI）
- `ant_etf_fund_flow`（ETH/BTC ETF 资金流）
- `ant_fund_flow`（交易所净流量和储备数据）

### MCP 服务安装

#### Antseer MCP

**Claude Code (CLI)**

```bash
claude mcp add --transport http --scope user antseer https://mcp.antseer.com/api/mcp
```

**OpenClaw / Claw**

在设置页面添加 MCP 服务器，填写：
- 名称：`antseer`
- URL：`https://mcp.antseer.com/api/mcp`
- 传输类型：`http`

**OpenCode**

在配置文件 `opencode.json` 中添加：

```json
{
  "mcpServers": {
    "antseer": {
      "type": "http",
      "url": "https://mcp.antseer.com/api/mcp"
    }
  }
}
```

**通用 MCP 客户端**

任何支持 MCP 协议的客户端均可接入，核心参数：
- MCP 端点：`https://mcp.antseer.com/api/mcp`
- 传输类型（Transport）：`http`
- 作用域（Scope）：`user`（推荐，跨项目共享）

安装完成后，**重启你的 Agent 客户端**以激活 MCP 服务。
