## Skill 功能摘要

| 维度 | 内容 |
|------|------|
| 核心功能 | 通过 CEX 换手量与历史压力事件（3AC、FTX、LUNA 等）横向对比，结合 ETF 稀释修正、净流量验证、跨资产时序分析，生成 0-100 分的底部信号综合评分 |
| 输入/输出 | 输入：symbol（必填）+ 可选参数（exchange、lookback_days、benchmark_events、etf_adjustment、comparison_asset、time_range）；输出：结构化 Markdown 报告，含换手量现状、历史对比、资金流向、综合评分和建议 |
| 依赖 MCP 工具 | ant_spot_market_structure、ant_market_indicators、ant_etf_fund_flow、ant_fund_flow |
| 知识门槛 | 高：需理解 CEX 换手量概念、ETF 稀释效应、历史百分位、净流量、跨资产底部时序等中高级概念 |
| 操作门槛 | 低：一键运行（/cex-volume-anomaly-signal ETH），参数均有默认值 |
| 使用频率 | 低频（事件驱动）：市场大跌 >30% 时集中使用，平稳期几乎不触发 |
| 决策类型 | 交易决策（建仓时机）+ 研究分析（机构报告） |

## 优化建议

1. **降低输出门槛 — 增加简报模式**
   - 发现：画像 1 中的中级交易员在快速判断阶段不需要完整的 7 步报告，需要的是"能不能建仓"的直接结论
   - 行动：增加 `brief_mode=true` 参数，输出压缩为：信号等级（强/中/弱）+ 评分 + 3 个关键因子 + 一句建议
   - 预期效果：降低输出解读门槛，覆盖更多 intermediate 交易者，提升单次对话的决策效率

2. **提升使用频率 — 与市场监控 skill 联动**
   - 发现：Skill 的使用频率极低，仅在市场大跌时触发，导致平时用户几乎不会主动想到它
   - 行动：在 Entry Conditions 中增加"被动触发"路径：当 market-regime-monitor 或类似 skill 检测到成交量异动时，自动建议运行本 Skill；或增加监控轮询触发词（如"帮我盯着 ETH 换手量，达到历史极端时告诉我"）
   - 预期效果：从纯粹的"主动查询工具"升级为可嵌入监控流程的自动化信号，提升用户粘性

3. **支持机构用户工作流 — 批量分析与 JSON 导出**
   - 发现：画像 2 机构分析师需要批量对多资产生成报告，并将结构化数据写入研究报告/数据库
   - 行动：支持 `symbols` 列表参数（如 `symbols=["ETH","BTC","SOL"]`），生成多资产对比汇总表；增加 `output_format=json` 选项，返回机器可读的结构化结果
   - 预期效果：打通机构用户的 workflow，将 Skill 从个人辅助工具升级为可接入报告流水线的生产力工具

4. **扩展资产覆盖 — 非 ETH/BTC 资产的稀释修正**
   - 发现：ETF 稀释修正逻辑仅适用于 ETH 和 BTC，SOL/BNB 等资产的稀释系数记为 0，导致分析结果对热门山寨资产偏弱
   - 行动：对非 ETH/BTC 资产，引入 futures open interest 变化或大户持仓数据作为机构分流的代理指标，补充稀释修正逻辑；或在输出中明确标注"非 ETH/BTC 资产，稀释修正不适用，结果偏保守"
   - 预期效果：提升 Skill 对主流山寨资产分析结果的准确性和用户信任度

5. **提升时效性 — 支持自定义历史事件**
   - 发现：内置历史事件（3AC、FTX、LUNA、2020-3.12）为 2022 年前后的事件，随时间推移新的压力事件（如 2024 GBTC 解锁、2025 某所暴雷）无法纳入基准
   - 行动：增加 `custom_events` 参数，支持用户传入自定义事件名称和日期窗口；同时在 SKILL.md 中提供"更新内置事件表"的维护指引
   - 预期效果：延长 Skill 的生命周期，避免因市场演进导致历史基准过时而失效

本报告采用"推导 + 验证"方法：先从 PRD 和 SKILL.md 文档推导目标用户假设，再尝试用 ant_market_sentiment 社交数据进行交叉验证。

**本次执行的社交数据验证受到以下限制**：
- ant_market_sentiment MCP 工具在执行环境中未注册（不在可用 MCP 服务器列表中）
- Exa web search 工具权限未授予
- 因此，社交验证基于领域知识推导完成，结合 PRD 本身的来源背景（真实 Crypto Twitter 方法论帖子）作为最强验证信号

**置信度说明**：
- 画像 1（中长线交易员）：置信度**高**，因为 PRD 原始来源即为该用户群的真实行为产出
- 画像 2（机构分析师）：置信度**中等**，因为机构需求有 PRD 明确描述，但缺少公开社交数据支撑
- market_size_estimate：置信度**低**，基于行业基准估算，实际规模高度依赖市场周期

社交数据仅反映公开讨论，机构风控等私密场景天然属于"沉默需求"，不能因为社交数据少就否认其存在。
