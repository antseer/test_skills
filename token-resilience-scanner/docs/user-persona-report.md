## Skill 功能摘要

| 维度 | 内容 |
|------|------|
| 核心功能 | 在大盘承压周期中，对候选代币计算双向 beta 韧性（上涨日超额 + 下跌日超额），按 T1/T2/弱势三档分层，生成标准化 Markdown 韧性周报 |
| 输入/输出 | 输入：代币列表（必填）+ 基准代币、时间窗口、流动性阈值（可选）；输出：完整 Markdown 周报（分层表格 + 叙事标签 + 下周观察建议） |
| 知识门槛 | 中等 — 需理解「超额收益」概念、区分上涨日/下跌日 alpha 的意义；不需要编程能力，但需要能解读百分比指标和 T1/T2 分层逻辑 |
| 使用频率 | 中频 — 每周收盘后定期执行（约 1 次/周）+ 大盘急跌事件驱动触发（约 1–2 次/月） |
| 决策类型 | 交易决策（仓位轮动选币）+ 研究分析（周报内容生成）+ 风控决策（弱势警示规避） |
| 依赖 MCP 工具 | `ant_spot_market_structure`（核心，价格和交易量数据）、`ant_market_sentiment`（可选，叙事情绪增强）、`ant_fund_flow`（可选，资金流向验证） |

## 优化建议

1. **降低首次使用门槛** — 画像 1（山寨轮动交易者）没有固定观察列表 → 增加预设候选池模板（如 `top30_altcoins`、`ai_narrative`、`defi_blue_chips`），一键调用无需手动枚举代币 → 预期覆盖更多中级交易者，减少因参数配置复杂性导致的流失。

2. **提升数据质量透明度** — 画像 3（量化策略开发者）对数据精度有高要求 → 在报告头部增加数据质量声明（sparkline 近似值 vs 精确收盘价），并优先探测精确历史价格接口 → 预期提升专业用户对分析结果的信任度，减少因精度质疑导致的使用放弃。

3. **增加异动日剔除选项** — 单日 +21% 等极端事件会拉偏 7 日均值，导致 T1 分层不准确 → 增加 `exclude_outliers=true` 参数，剔除超过 2 倍标准差的异动日后重新计算，并在报告中对比展示 → 预期帮助所有用户区分真实持续性韧性与单日事件驱动，提升分层准确率。

4. **增加 JSON 结构化输出模式** — 画像 3 需要将 resilience_score 导入回测框架，当前仅 Markdown 输出无法支持 → 增加 `output_format=json` 选项，输出每个代币的完整韧性指标数组 → 预期覆盖量化开发者用户群，使 Skill 可作为数据管道组件而非仅用于人工阅读报告。

5. **社交情绪作为弱势信号备用维度** — `ant_fund_flow` 山寨币覆盖度不确定，弱势警示的「资金流出」维度可能无法获取 → 在弱势警示章节调用 `ant_market_sentiment coin_detail` 补充社交热度趋势，从价格韧性 + 情绪两个维度交叉验证弱势信号 → 预期减少弱势警示的误报/漏报，对三类用户画像均有价值。

## 方法论说明

本报告采用「推导 + 验证尝试」方法：先从 PRD 文档和 SKILL.md 推导目标用户假设，再尝试用 ant_market_sentiment 社交数据进行交叉验证。由于 MCP 工具在当前执行环境中不可用，社交验证部分未能完成，所有画像置信度标注为「推断」或「部分验证」。

实际需求的最强证据来自 PRD 本身：推文原作者 @Guolier8 每周手动整理此类分析并公开发布，其追随者的互动行为直接反映了目标用户群的真实存在和需求强度。本 Skill 本质上是对该手动分析流程的自动化复现，需求验证已内嵌于 PRD 来源中。

社交数据仅反映公开讨论，无法覆盖专业研究者和量化开发者的私下工作流需求（即「沉默需求」）。
