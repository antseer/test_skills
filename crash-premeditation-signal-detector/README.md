# crash-premeditation-signal-detector

崩盘预谋模型信号检测 — 用合约数据识别主力高位出货+空单布局的砸盘前兆信号

---

## 这个工具做什么？

你在 BTC/ETH 价格上涨时，想知道"这次拉升是真多头还是主力在出货"——这个工具帮你回答这个问题。

背景：有一种经典的主力砸盘操作，三步走：
1. **出货阶段**：主力悄悄把多单平掉，同时挂大量买单（托单）让价格看起来很稳，不让价格跌
2. **诱多阶段**：散户和量化做市商（"电子盘"）因为价格稳定而进场做多，成为主力空单的对手盘
3. **砸盘阶段**：主力撤掉托单、叠加空单顺势砸盘，触发连锁爆仓，收割完毕

这个工具的识别逻辑：**正常牛市拉升时 OI（合约持仓量）是增加的；而主力出货时，价格被托着涨但 OI 在持续下降**。在这个核心信号基础上，再用 7 个维度交叉验证，给出 4 级信号评级。

方法论来源：@wuk_Bitcoin，2026-02-01 推文。

---

## 怎么用？

```
/crash-premeditation-signal-detector BTC
/crash-premeditation-signal-detector ETH 8
/crash-premeditation-signal-detector BTC 4 --oi_decline_threshold_pct=3.0
```

**参数说明：**

| 参数 | 必填 | 默认值 | 说明 |
|------|------|--------|------|
| symbol | 是 | — | 交易对，大写，如 BTC、ETH |
| lookback_hours | 否 | 6 | 回溯分析窗口（小时），推荐 4-12 |
| timeframe | 否 | 1h | 数据时间粒度（1h 或 4h） |
| oi_decline_threshold_pct | 否 | 5.0 | OI 下降幅度触发阈值（%） |
| price_rise_threshold_pct | 否 | 1.0 | 价格上涨最低幅度（%），过滤横盘噪声 |

**小技巧：**
- 作者建议用 1H 级别，5m/15m 粒度无法确认主力真实意图
- lookback_hours 设 4-8 为佳，太短容易误判，太长信号失去时效性
- 如果市场波动较小，可把 `price_rise_threshold_pct` 调低到 0.5

## 什么时候用？什么时候不适合？

**适合：**
- BTC、ETH 等主流合约品种出现异常拉升时
- 定期巡检（建议每小时对主流品种跑一次）
- 持有大量多单，想做风控排查的时候
- 看到推特上有人说"这波涨势不健康"，想量化验证

**不适合：**
- 小市值山寨币（OI 和 Smart Money 数据覆盖有限）
- 想知道"砸盘会在几点发生"——这个工具只能识别"正在布局阶段"，无法预测精确时间
- 现货市场分析（此工具专注合约/衍生品）
- 当 Smart Money 数据显示中性时，不能单独依赖此工具下结论

**局限性的诚实说明：**
- 工具无法检测订单簿中的托单行为（缺乏 L2 深度数据）
- "价格涨 + OI 跌"也可能是市场自然去杠杆，和主力砸盘很难区分，需结合 Smart Money 维度辅助判断
- OI 数据有 1-5 分钟延迟
- 资金费率每 8 小时结算一次，中间时段变化反映不够及时

## 前置依赖

### 为什么需要装 MCP？

这个工具需要实时的链上/市场数据才能工作。MCP（Model Context Protocol）服务器是数据来源 — 把它理解成"给 Claude 接上数据管道"。不装的话，Claude 拿不到数据，分析就跑不起来。

### MCP 服务安装

#### ant-on-chain-mcp

**Claude Code (CLI)**
```bash
claude mcp add --transport http --scope user ant-on-chain-mcp https://ant-on-chain-mcp.antseer.ai/mcp
```

**OpenClaw / Claw**

在设置页面添加 MCP 服务器，填写：
- 名称：`ant-on-chain-mcp`
- URL：`https://ant-on-chain-mcp.antseer.ai/mcp`
- 传输类型：`http`

**OpenCode**

在配置文件 `opencode.json` 中添加：
```json
{
  "mcpServers": {
    "ant-on-chain-mcp": {
      "type": "http",
      "url": "https://ant-on-chain-mcp.antseer.ai/mcp"
    }
  }
}
```

**通用 MCP 客户端**

任何支持 MCP 协议的客户端均可接入，核心参数：
- MCP 端点：`https://ant-on-chain-mcp.antseer.ai/mcp`
- 传输类型（Transport）：`http`
- 作用域（Scope）：`user`（推荐，跨项目共享）

安装完成后，**重启你的 Agent 客户端**以激活 MCP 服务。
