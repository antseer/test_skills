## Skill 功能摘要

| 维度 | 内容 |
|------|------|
| 核心功能 | 监控交易所仓位数据的极端变化（30天变化率超阈值），基于历史回测胜率生成反向操作信号 |
| 输入/输出 | 输入：代币symbol（必填）+ 7个可选参数（交易所、仓位类型、阈值等）；输出：结构化信号报告（当前信号、历史胜率、信号强度、辅助验证） |
| MCP 工具依赖 | ant_futures_market_structure（OI/多空比/资金费率）、ant_spot_market_structure（价格）、ant_futures_liquidation（爆仓）、ant_fund_flow（储备/鲸鱼转账） |
| 知识门槛 | 高 -- 需理解OI、保证金仓位、滚动变化率、皮尔逊相关系数、滞后交叉相关、统计胜率、反向交易逻辑 |
| 操作门槛 | 中 -- 最简用法仅需1个参数，但8个可选参数和输出中的统计指标需要一定解读能力 |
| 使用频率 | 中频（每日巡检）+ 低频事件驱动（极端信号每月0-2次） |
| 决策类型 | 交易决策 -- 直接输出做空/做多/持有建议及历史胜率 |

## 优化建议

1. **降低知识门槛：增加简报模式** -- 用户需理解OI、统计胜率等多个概念才能有效解读输出。增加"简报模式"，第一行直接给出结论（如"当前BTC无异常信号，距做空阈值还差3.2%"），首次使用时附30字内的术语速览。预计用户覆盖面扩大30-50%。

2. **提升使用频率：增加渐进告警和复合信号** -- 极端信号每月仅0-2次，大部分时间输出"无信号"，用户粘性不足。增加"距离阈值距离"的渐进告警（70%阈值预警），并与高频skill联动（资金费率异常+仓位接近阈值=复合预警）。将有效输出频率从每月0-2次提升到每周1-2次。

3. **增强数据透明度：标注数据源差异** -- 核心数据源Bitfinex Margin Longs在MCP中无覆盖，默认使用OI/多空比替代。在输出中明确标注当前数据源及其与原始方法论的差异，增加数据源可靠性评分。提升专业用户信任度。

4. **降低操作门槛：提供预设模式** -- 8个可选参数对非量化用户形成障碍。提供3个预设模式：快速巡检（仅需symbol）、标准分析（symbol+exchange+position_type）、深度研究（全参数）。90%场景只需一个参数，提升新用户转化率。

5. **增加主动监控能力** -- 当前为被动查询工具，缺乏持续监控。增加"监控模式"：用户设置symbol和阈值，skill每日自动检查并在信号触发时推送告警。从被动工具升级为主动监控系统，匹配"不想错过极端信号但不想天天手动检查"的核心痛点。
