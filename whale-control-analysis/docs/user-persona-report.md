## Skill 功能摘要

| 维度 | 内容 |
|------|------|
| 核心功能 | 分析代币 Top Holders 的链上持仓，通过地址聚类识别同一实体的多钱包控盘模式，计算真实控盘比例 |
| 输入 | 代币符号 + 所在链（必填），合约地址、分析数量、时间窗口等（可选） |
| 输出 | 结构化 JSON 报告：实体聚类、控盘比例、HHI 指数、风险等级、做市商身份关联、合约市场关联分析 |
| MCP 工具依赖 | ant_meme, ant_token_analytics, ant_address_profile, ant_futures_market_structure |
| 知识门槛 | **高** -- 需理解 Holder 聚类、资金来源分析、HHI 指数、OI/资金费率、控盘度概念 |
| 操作门槛 | **中** -- 输入简单（symbol + chain），但输出 JSON 结构复杂，需要二次解读 |
| 使用频率 | **低频/事件驱动** -- 代币出现异常价格走势时触发 |
| 决策类型 | 风控决策 + 研究分析（判断是否存在操纵风险，辅助投资决策） |

## 优化建议

### 1. 增加简报模式，降低输出解读门槛

**发现**: 用户门槛高 -- Skill 输出是复杂的 JSON 结构，需理解 HHI 指数、控盘比例、置信度等专业概念。合约交易者画像（画像 2）更需要直接结论而非原始数据。

**建议**: 增加 `output_mode` 参数，支持 `"brief"` 模式。简报模式输出 3-5 句自然语言摘要，直接回答：(1) 该代币是否被高度控盘？(2) 控盘实体是谁？(3) 对交易者意味着什么风险？

**预期效果**: 将目标用户从约 2,000 名专业分析师扩展到约 10,000 名中级交易者。

---

### 2. 增加监控模式，提升使用频率

**发现**: 当前为事件驱动（低频），用户仅在代币异常时才触发分析，缺乏持续价值。

**建议**: 增加代币监控列表功能，Skill 定期扫描 Top Holders 变化，当控盘度突破阈值（如从 40% 升至 60%）时主动推送告警。可与 regime 监测服务联动，将控盘度变化作为一种 regime signal。

**预期效果**: 从「被动工具」升级为「主动监控」，将使用频率从每月数次提升到每日被动接收，增加用户粘性。

---

### 3. 增加快速扫描模式，降低执行成本

**发现**: 完整分析需 200+ 次 MCP 调用（50 地址 x 多维查询），耗时较长且可能触发 rate limit。

**建议**: 增加 `scan_mode` 参数，支持 `"quick"`（Top 10 地址）和 `"full"`（Top 50 地址）。快速扫描先分析 Top 10，若已显示高控盘度（>60%）则直接输出初步结论，用户可选择继续深度分析。

**预期效果**: 常见场景执行时间缩短 80%，降低 MCP 调用成本，提升交互体验。

---

### 4. 扩展 Solana 和 Base 链覆盖

**发现**: 当前以 EVM 链（ethereum）为默认，但 meme 币热点频繁出现在 Solana（pump.fun）和 Base 上，而庄家控盘在 meme 币中最为常见。

**建议**: 在 SKILL.md 中明确 Solana、Base 链的数据可用性和降级策略。优先确保这三条链的完整覆盖。

**预期效果**: 覆盖 Solana meme 币场景，预计扩大 30-40% 的可用场景。

---

### 5. 增加操纵模式识别，形成竞品差异化

**发现**: Bubblemaps 和 Arkham 已提供持仓分析功能，本 Skill 的差异化需要更清晰。

**建议**: 在输出中增加「操纵模式识别」模块：自动匹配已知模式（如「现货控盘 + 合约收割」「归集后砸盘」「分批建仓后拉盘」），给出模式名称、风险等级和历史案例参考。

**预期效果**: 从「数据工具」升级为「策略工具」，形成与 Arkham/Bubblemaps 的明确差异化，提升对专业交易者的吸引力。
