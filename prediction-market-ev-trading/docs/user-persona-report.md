## Skill 功能摘要

| 维度 | 内容 |
|------|------|
| 核心功能 | 多数据源交叉验证构建独立概率模型，与 Polymarket 等预测市场定价对比发现 +EV 交易机会，Kelly 公式仓位管理 + 自校准闭环 |
| 输入/输出 | 输入: 市场品类、数据源列表、Kelly 参数、EV 阈值；输出: +EV 交易机会列表、仓位建议、校准统计、策略健康度报告 |
| 知识门槛 | **高** -- 需理解正期望值(EV)计算、Kelly 公式、Brier Score、多源交叉验证、概率校准度等量化概念 |
| 操作门槛 | **中高** -- 需配置外部 API（Polymarket、气象数据源等），指定品类和数据源参数 |
| 使用频率 | 中频 -- 每日扫描 + 事件驱动（重大结算前加密扫描） |
| 决策类型 | 交易决策（直接影响下注金额和方向） |
| MCP 覆盖度 | **极低** -- 核心数据（预测市场赔率、天气数据、结算结果）全部需外部 API；仅 crypto 品类可部分使用现有 MCP |

## 优化建议

### 1. 优先打造 Crypto 预测市场 MVP

**发现**: MCP 覆盖度极低（核心数据全部需外部 API），但 crypto 品类可部分利用现有 MCP 工具（ant_spot_market_structure、ant_market_indicators 等）。

**改动**: 将 crypto 预测市场（如 "BTC 月底是否突破 $100k"）作为 MVP 品类，利用现有 MCP 工具实现价格预测 + 市场赔率对比。天气、体育、政治品类作为 v2 扩展。

**预期效果**: 在不增加外部 API 依赖的情况下交付可运行的 MVP，验证核心逻辑（EV 计算 + Kelly 仓位）后再扩展品类。

### 2. 增加简报模式降低知识门槛

**发现**: 用户需理解 EV、Kelly、Brier Score 等多个量化概念。知识门槛将大量预测市场参与者（非量化背景）排除在外。

**改动**: 新增 `output_mode=summary` 参数，输出简化的交易建议（推荐/不推荐 + 置信度 + 1句理由），隐藏底层数学推导。保留 `output_mode=full` 给高级用户。

**预期效果**: 潜在用户群从"量化交易者"扩展到"普通预测市场参与者"，预估覆盖范围扩大 2-3 倍。

### 3. 内置基线校准数据改善首次体验

**发现**: 自校准需要 30 次历史数据，新用户首次运行时无校准数据，输出的置信度和建议可靠性未知。

**改动**: 预置各品类的基线校准参数（如天气市场的数据源历史准确率、crypto 技术指标的预测胜率基准），让首次运行也有合理的默认权重。标注"使用基线参数，建议积累 30 次数据后切换为自校准模式"。

**预期效果**: 首次使用即可获得有参考价值的输出，缩短用户从"试用"到"信任"的周期。

### 4. 整合 Polymarket API 作为内置数据源

**发现**: Polymarket CLOB API 是公开可用的，但当前 Skill 要求用户自行提供市场数据。这是最大的使用摩擦点。

**改动**: 在 Step 1（市场扫描）中内置 Polymarket CLOB API 调用（`https://clob.polymarket.com`），自动获取活跃市场列表和实时赔率。用户只需指定品类即可开始分析。

**预期效果**: 将操作门槛从"需要自行接入 API 并提供数据"降低为"选择品类即可运行"，大幅提升开箱即用体验。

### 5. 增加英文输出支持

**发现**: 预测市场讨论以英文社区为主（Polymarket 用户主要为英文用户），但 Skill 文档和输出均为中文。

**改动**: 增加 `language=en` 参数支持英文输出，默认中文。策略报告中的关键术语（EV、Kelly fraction、Brier Score）已保持英文，进一步扩展到完整英文报告模板。

**预期效果**: 覆盖英文为主的 Polymarket 原生用户群，提升 Skill 在国际 crypto 社区的适用性。

## 方法论说明

本报告采用"推导 + 验证"方法：先从 Skill 文档和 PRD 推导目标用户假设，
再尝试用 ant_market_sentiment 社交数据进行交叉验证。

**本次分析的限制:**
- ant_market_sentiment MCP 工具在当前运行环境不可用，社交验证环节无法执行
- 社交数据相关的发现和判断基于分析者的领域知识推断，置信度低于有数据支撑的版本
- 建议在 MCP 工具可用时重新运行社交验证步骤，更新画像的验证状态

**即便如此，以下结论具有较高可靠性（基于 PRD 和 SKILL.md 文档分析）:**
- Skill 的核心用户是具备量化背景的预测市场交易者（画像 1）
- MCP 覆盖度不足是当前最大的落地障碍
- 知识门槛和操作门槛限制了潜在用户群的规模
- Crypto 品类是最适合作为 MVP 的方向（利用现有 MCP 工具）
