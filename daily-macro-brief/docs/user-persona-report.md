## 优化建议

1. **降低知识门槛，增加简报模式** — 当前 5-section 完整报告对新用户门槛偏高，需同时掌握宏观经济学和加密市场微观结构知识。建议增加 `brief_mode=simple` 参数，输出仅包含 Risk-On/Off 评分、一句话主导叙事和 3 条最重要信号，隐藏详细数据快照和历史类比。预期效果：将潜在用户从'熟练宏观交易者'扩展到'有宏观意识的中级用户'，覆盖人群扩大 3-5 倍。

2. **补全外部数据缺口，增加 web fallback** — 原油、标普 500、美债收益率（10Y/2Y）、VIX 和高收益利差共 6 个关键宏观指标无 MCP 覆盖（约 35% 的跨资产分析依赖用户自行提供数据），严重影响产品完整性。建议增加 `external_data_mode=web` 参数，通过 web search 自动获取 Yahoo Finance（CL=F、^VIX、^GSPC）和 FRED（DGS10、DGS2、BAMLH0A0HYM2）数据，并在输出中明确标注数据新鲜度。预期效果：外部数据覆盖率从 ~65% 提升至 ~90%，报告可信度显著提升。

3. **针对 KOL 场景增加多格式输出** — KOL 用户需要将简报格式化为推文串或 Telegram 消息后再发布，当前纯 markdown 输出增加了额外摩擦。建议增加 `output_format` 参数：支持 `markdown`（默认）/ `tweet_thread`（5-8 条推文串，每条 ≤280 字符）/ `telegram`（Telegram 兼容 markdown，含方向 emoji 标注）。预期效果：显著降低 KOL 用户的内容发布摩擦，提升该用户群日活频率。

4. **增加事件驱动触发模式** — 当前 Skill 主要针对每日定时触发场景设计，但用户在重大宏观事件（FOMC 决议、CPI 数据、非农就业）后有强烈的临时分析需求。建议增加 `--event` 参数（如 `/daily-macro-brief --event=FOMC`），自动聚焦该事件的传导链分析，并对比历史同类事件（如'过去 5 次 FOMC 加息后 BTC 48h 表现'）。预期效果：将使用场景从'纯日常工具'扩展到'事件驱动工具'，提升高价值时刻的使用频率。

5. **优化双语支持，增加语言自动检测和数据源本地化** — 中文和英文用户群体在内容偏好和工具习惯上差异显著，当前 `language=zh` 为硬编码默认值，对英文用户不友好。建议将语言默认值改为自动检测（根据用户 Claude 会话语言偏好），同时为英文模式提供英文数据源建议（Blockworks、The Block RSS），为中文模式提供中文数据源建议（律动、PANews、链闻）。预期效果：降低国际化用户配置负担，英文用户群体占比预计从 ~20% 提升至 ~40-50%。

本报告采用"推导 + 文档验证"方法：从 PRD 文档和 SKILL.md 的功能描述、目标用户定义、分析框架设计中推导目标用户假设，并用 PRD 内部的 sample brief 和工具设计细节进行内部交叉验证。

社交数据无法覆盖的场景：机构/基金场景（私密需求，不在社交平台讨论）。该画像的置信度标注为 medium，建议通过用户访谈补充验证。

**重要局限**: 宏观话题的社交讨论热度随市场周期显著波动（熊市/牛市对宏观关注度差异大），本报告是时间快照，实际市场环境变化可能影响用户需求强度。
