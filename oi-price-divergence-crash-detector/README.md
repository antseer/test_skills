# oi-price-divergence-crash-detector

OI 背离崩盘预警 — 用合约持仓量数据检测主力高位平多布空的做空信号

---

## 这个工具做什么？

你在看合约行情时，发现价格没怎么跌、甚至还在涨，但盘感告诉你"这不对劲"——这个工具帮你把"不对劲"量化出来。

它的逻辑来自交易员 @wuk_Bitcoin 描述的**主力预谋崩盘三阶段模型**：

1. **洗盘期**：主力在高位悄悄平多单，但用挂单托住价格，让你以为"行情还在"。这时 OI（Open Interest，合约未平仓量——理解成"市场上有多少人开着合约"）会连续数小时下降，但价格横盘甚至微涨。
2. **布空期**：多单清完了，主力继续托价，吸引散户和量化算法入场做多（给主力当对手盘），同时自己悄悄建空单。
3. **砸盘期**：撤掉托价挂单，顺势砸盘 + 持续加空，引爆连锁爆仓。

这个工具做三件事：
1. 拉取过去 6 小时（可调）的 1H 价格和 OI 数据，识别"价格横盘/微涨 + OI 连续下降"的背离区间
2. 用资金费率、散户多空比、Smart Money 动向、爆仓热力图、链上鲸鱼预警做多维度交叉验证
3. 输出综合预警等级（高危 / 中等 / 低风险）和量化得分

---

## 怎么用？

```
/oi-price-divergence-crash-detector BTC
/oi-price-divergence-crash-detector ETH 1h 8
/oi-price-divergence-crash-detector SOL 2h 12
```

| 参数 | 说明 | 必填 | 默认值 |
|------|------|------|--------|
| `symbol` | 合约交易对，如 BTC、ETH、SOL | 是 | — |
| `interval` | K 线时间粒度 | 否 | `1h` |
| `lookback_hours` | 回溯分析时间窗口（小时） | 否 | `6` |
| `oi_decline_threshold_pct` | 单根 K 线背离触发阈值（%） | 否 | `3.0` |
| `exchange` | 指定交易所或聚合 | 否 | `aggregated` |

## 什么时候用？什么时候不适合？

**适合：**
- BTC/ETH/SOL 等主流合约，价格走势感觉"被人托着"但 OI 在悄悄缩
- 行情处于高位震荡、短期不跌但上涨动能不足时
- 量化巡检：每 1-4 小时自动检查主流品种有无异动信号
- 风控场景：多头仓位持有中，想确认是否需要减仓

**不适合：**
- MEME 等低流动性币种（OI 基数小，随机波动就能触发阈值）
- 现货市场（本工具仅分析合约数据）
- 想精确预测崩盘时间点（信号出现到真正砸盘可能间隔数小时到数天）
- 需要订单簿深度数据（托价挂单的直接证据，当前 MCP 不覆盖）

## 前置依赖

### 为什么需要装 MCP？

这个工具需要实时的链上/市场数据才能工作。MCP（Model Context Protocol）服务器是数据来源 — 把它理解成"给 Claude 接上数据管道"。不装的话，Claude 拿不到数据，分析就跑不起来。

### MCP 服务安装

#### ant-on-chain-mcp

**Claude Code (CLI)**
```bash
claude mcp add --transport http --scope user ant-on-chain-mcp https://ant-on-chain-mcp.antseer.ai/mcp
```

**OpenClaw / Claw**

在设置页面添加 MCP 服务器，填写：
- 名称：`ant-on-chain-mcp`
- URL：`https://ant-on-chain-mcp.antseer.ai/mcp`
- 传输类型：`http`

**OpenCode**

在配置文件 `opencode.json` 中添加：
```json
{
  "mcpServers": {
    "ant-on-chain-mcp": {
      "type": "http",
      "url": "https://ant-on-chain-mcp.antseer.ai/mcp"
    }
  }
}
```

**通用 MCP 客户端**

任何支持 MCP 协议的客户端均可接入，核心参数：
- MCP 端点：`https://ant-on-chain-mcp.antseer.ai/mcp`
- 传输类型（Transport）：`http`
- 作用域（Scope）：`user`（推荐，跨项目共享）

安装完成后，**重启你的 Agent 客户端**以激活 MCP 服务。
